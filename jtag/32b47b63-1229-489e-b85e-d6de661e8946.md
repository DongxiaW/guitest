## JTAG state encoding (background)

`jtagState` is a 4-bit code; `J_state_ascii` is the same state encoded as
14 ASCII characters (112 bits) so the bus is human-readable in GTKWave.

| code | state | meaning |
|---|---|---|
| 0 | `testLogicReset` | reset |
| 1 | `runTest` | idle |
| 2 | `selectDR` | choose DR path |
| 3 | `captureDR` | capture into DR |
| **4** | **`shiftDR`** | **shift DR through TDI/TDO** ← target |
| 5 | `exit1DR` | leave shift |
| 6 | `pauseDR` | pause shifting |
| 7 | `exit2DR` | resume / update |
| 8 | `updateDR` | latch DR |
| 9–15 | `selectIR` … `updateIR` | IR-path mirror states |

See [jtag.v](jtag.v) for the RTL and [TASK_GTKWAVE.md](TASK_GTKWAVE.md) for the
human-facing benchmark write-up.

## Evaluator

`func: check_include_exclude` runs a base64-embedded Python script
(`/bin/bash -c "echo <b64> | base64 -d | python3"`) that parses the saved
`jtag.gtkw` and prints one `OK_<name>` / `FAIL_<name>` line per check. The task
passes iff **all `include` tokens appear** and **no `FAIL_` token appears**.

### What the script checks

| Check | OK token | What it looks for in `jtag.gtkw` |
|---|---|---|
| Signals present | `OK_treset`, `OK_tck`, `OK_tms`, `OK_jtagState`, `OK_ascii_signal` | substrings `tb.treset`, `tb.tck`, `tb.tms`, `tb.jtagState`, `J_state_ascii` |
| `jtagState` is Decimal | `OK_dec_fmt` | nearest preceding `@<hex>` directive has `TR_DEC` bit (`0x4`) set |
| `J_state_ascii` is ASCII | `OK_ascii_fmt` | nearest preceding `@<hex>` directive has `TR_ASCII` bit (`0x800`) set |
| Named marker exists | `OK_marker_named` | regex `\[markername_\d+\]\s*firstShiftDR` |
| Marker is past reset | `OK_marker_time_post_reset` | at least one value on the `*<scale> …` line is > 30 (gtkw ticks) |
| Signal ordering | `OK_order` | first occurrences of the five names appear in the spec order |

### Trace-flag bits (gtkwave)

From [analyzer.h](https://github.com/gtkwave/gtkwave/blob/master/src/analyzer.h)
(`TraceEntFlagBits` enum, current master):

```
TR_HIGHLIGHT_B = 0   → 0x001
TR_HEX_B       = 1   → 0x002
TR_DEC_B       = 2   → 0x004    ← jtagState should have this
TR_BIN_B       = 3   → 0x008
TR_OCT_B       = 4   → 0x010
TR_RJUSTIFY_B  = 5   → 0x020
TR_INVERT_B    = 6   → 0x040
TR_REVERSE_B   = 7   → 0x080
TR_EXCLUDE_B   = 8   → 0x100
TR_BLANK_B     = 9   → 0x200
TR_SIGNED_B    = 10  → 0x400
TR_ASCII_B     = 11  → 0x800    ← J_state_ascii should have this
```

Older gtkwave headers omit `TR_HIGHLIGHT`, which off-by-ones every bit and
makes online references say `TR_DEC=0x2 / TR_ASCII=0x400`. The current source
(and this evaluator) use `0x4 / 0x800`.

### Decoded Python script

```python
import re, sys
p = "/home/user/edatest/jtag/jtag.gtkw"
try:
    s = open(p).read()
except Exception:
    print("FAIL_NO_FILE"); sys.exit()

out = []
def chk(n, c): out.append(("OK_" if c else "FAIL_") + n)

# 1. Signal presence
chk("treset",       "tb.treset"      in s)
chk("tck",          "tb.tck"         in s)
chk("tms",          "tb.tms"         in s)
chk("jtagState",    "tb.jtagState"   in s)
chk("ascii_signal", "J_state_ascii"  in s)

# 2. Format directives -- walk back to nearest @<hex> line before each signal
def fmt_before(sig):
    lines = s.splitlines()
    for i, ln in enumerate(lines):
        if sig in ln and not ln.startswith("@"):
            for j in range(i - 1, -1, -1):
                if lines[j].startswith("@"):
                    try: return int(lines[j][1:], 16)
                    except: return None
                if lines[j] and lines[j][0] not in "-[":
                    break
    return None

fj = fmt_before("tb.jtagState")
fa = fmt_before("J_state_ascii")
# TR_DEC = 0x4, TR_ASCII = 0x800 (gtkwave TraceFlag bits)
chk("dec_fmt",   fj is not None and (fj & 0x4)   != 0)
chk("ascii_fmt", fa is not None and (fa & 0x800) != 0)

# 3. Named marker
chk("marker_named",
    re.search(r"\[markername_\d+\]\s*firstShiftDR", s) is not None)

# 4. Marker positioned past reset
m = re.search(r"^\*([0-9.\-]+)\s+([0-9\- ]+)$", s, re.M)
if m:
    try:
        ticks = [int(x) for x in m.group(2).split()]
        chk("marker_time_post_reset", any(t > 30 for t in ticks))
    except Exception:
        chk("marker_time_post_reset", False)
else:
    chk("marker_time_post_reset", False)

# 5. Signal ordering matches task spec
order = ["tb.treset", "tb.tck", "tb.tms", "tb.jtagState", "J_state_ascii"]
pos = [s.find(x) for x in order]
chk("order", all(p >= 0 for p in pos) and pos == sorted(pos))

for r in out: print(r)
```

## Known caveats

- **`fmt_before` is strict about layout.** It only finds the format directive
  when an `@<hex>` line sits immediately above the target signal (other signal
  lines in between cause it to give up). That holds here because both
  `jtagState` and `J_state_ascii` get their own format directive — but reusing
  the helper for signals that inherit a group's `@` will silently return
  `None`.
- **`marker_time_post_reset` is lenient.** `t > 30` is in gtkw native ticks
  (typically picoseconds), not nanoseconds as the comment suggests. Effectively
  it just verifies a marker/cursor was placed somewhere past the very start.
- **Empty exception handler:** missing `jtag.gtkw` prints `FAIL_NO_FILE` and
  exits cleanly (code 0). The `exclude: ["FAIL_"]` rule still catches it.
